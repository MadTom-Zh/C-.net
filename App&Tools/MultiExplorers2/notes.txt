

功能（基本同V1.0）：
基本和Windows自带的浏览器一样，
但只是只用于浏览、复制、移动、重命名，不提供搜索功能；
支持鼠标拖动、快捷键；
对外读写时，当磁盘被占用，暂时拒绝读取或写入，以保证最佳读写效率（当然工具之外的读写状态不考虑）；


优势：
复制、移动文件时，此软件会按照设备的占用情况来决定是否进行传输作业，
例如3磁盘之间互传文件，则同一块磁盘不会进行多个文件同时写入（多分区时也算1磁盘），且不会同时进行读和写；
在对网络位置读写时，按主机名字（ip），来断定为一个磁盘；


其他：
后台获取网络位置，启动时不卡界面；


界面：
选中文件、文件夹时，在状态栏显示信息（文件夹内容信息不显示，节省性能）
包括：名称、尺寸、属性、修改时间； 选中多个内容时，显示选中了什么；

操作：
支持从外部拖拽进入，（（（（但无法拖拽到外界；））））？
支持内部双方拖拽；
支持快捷键复制、粘贴；

事项：
如果向外界拖拽，则无法由程序控制传输过程；
如果向外界使用粘贴快捷键，一样程序不控制过程；


顶部工作条：
打开文件浏览器（explorer），磁盘工作状态检查（显示当前各磁盘的读写状态），帮助（打开帮助文档html）



作业图标：
光标放在任务按钮上，显示此任务的动作描述；


鼠标点击显示提示，或等待若干秒显示提示；
图标为长图，布局为左右分别源盘图标和目标盘图标，中间是如“C-->D”或“C-|D”；
图标显示磁盘状态，如果正在传输则 中间为如“C+>D”，“C->D”，否则“C||D”；
传输状态时，左右图标均显示为原始图标，如果等待，则显示盘的状态，如上面有红色向下箭头意思为正在写入，或绿色向上，为正在读取；



作业执行顺序和事项：
作业默认顺序排列，不相干的作业（如第二作业的文件内容不涉及第一作业的内容），且磁盘互不影响（1磁盘不会同时多写入，多读取，或同时读和写），则这些作业将并行执行；
正在执行的作业显示为复制-浅绿色，移动-蓝色，删除-橙色，边框为对应表面颜色的加深；暂停或等待作业浅灰色，边框颜色不变；
双击图标来暂停，或开始/继续，如要取消，右击按钮，并选择取消；
* 删除默认为丢入回收站，如果是网络删除（不支持回收站），则会给出永久删除提示；
* 不支持撤销操作，如出现误操作，请手动还原；
* 设置中的立即执行操作量大小阈值，如果复制、异分区移动，当小于阈值时，将立即执行；
* 同分区移动（或重命名）、删除，将被立即执行；



日志记录类型：
常规行为，程序启动，复制、剪切、粘贴，查看属性，执行菜单命令，布局保存、应用，样式更改，程序关闭；
+程序报错，捕捉任何未处理的报错；
存储访问，读取磁盘清单，读取共享目录、读取文件夹等；
存储插拔，当有usb等存储设备插入或拔出时；
+传输作业，记录作业的基本事件（含作业号），作业的完成；
传输细节，（作业号之下）每个文件操作的细节，包括文件大小，消耗时间等；
+传输报错，（作业号之下）当不能读取源、写入失败等时；


测试命令
Bandizip.exe <command> [<switches>...] <archive> [<files>...] [<path_to_extract>]
* 详解见 https://www.bandisoft.com/bandizip/help/parameter/

压缩多个文件到一个压缩包：
Bandizip.exe c -l:9 -r "C:\Temp\test.zip" "C:\Temp\100 eg.txt" "C:\Temp\a13"
             | |    |  |                  + 要打包的文件/文件夹
             | |    |  + 被创建的压缩文件
             | |    + 递归子目录，多层目录的内容也被打包；
             | + 压缩等级0-9，9压缩率最大；
             + 创建，存在则覆盖
